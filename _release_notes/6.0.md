<b>\*\*\* Squeak 6.0 Release Notes \*\*\*</b>

We are happy to release the next version of Squeak shortly after its 25th anniversary. This is the first Squeak that supports high-resolution displays on all major platforms and offers a flexible <a href="code://RealEstateAgent class&gt;&gt;#scaleFactor">UI scale factor</a> through improved support for <a href="code://TheWorldMainDockingBar instance fontSizeSummary">TrueType fonts</a>. Bundled with the <a href="https://github.com/OpenSmalltalk/opensmalltalk-vm/releases/tag/202205110711">latest OSVM release</a>, the image now uses the new <a href="code://(Preferences preferenceAt: #&apos;CompiledCode&gt;&gt;preferredBytecodeSetEncoderClass&apos;) open
">SistaV1 bytecode set</a> by default. We changed the <a href="code://SystemReporter open">image-format version</a> so that older VMs will not crash on Sista-specific bytecodes. Furthermore, we made many improvements to our programming tools to provide a better programming experience for code writing, object inspection, and process debugging.<font color="#FF0000">
</font>
Please find below a detailed listing of all the changes, which includes notes on <b>Major Deprecations</b>, <b>Known Issues</b>, and <b>Compatibility Notes</b> at the bottom.

With Sista, blocks are represented by <a href="code://TextLink new classAndMethod: &apos;CompiledBlock Comment&apos;; actOnClickFor: Model new">CompiledBlocks</a> and instantiated as <a href="code://TextLink new classAndMethod: &apos;FullBlockClosure Comment&apos;; actOnClickFor: Model new">FullBlockClosures</a>, the stability of which has been improved in the course of this switch (fixed bugs in <a href="code://TextLink new classAndMethod: &apos;FullBlockClosure =&apos;; actOnClickFor: Model new">FullBlockClosure &gt;&gt; \#=</a> and <a href="code://TextLink new classAndMethod: &apos;InstructionStream scanForInstructionSequence:&apos;; actOnClickFor: Model new">InstructionStream &gt;&gt; \#scanForInstructionSequence:</a>). <b>Known Issues</b>
- Lookbehinds in regular expression are not yet considered stable. <a href="code://TextLink new classAndMethod: &apos;RxParserTest testOptionalLookbehind2&apos;; actOnClickFor: Model new">Inserting them into a nullable closure currently causes a MessageNotUnderstood error.</a> Also, matching is not optimized and currently has a quadratic runtime complexity.

<b>Compatibility Notes</b>
- SqueakSSL is not supported under Ubuntu 16.04 LTS.
- All code literals are read-only with the new bytecode set SistaV1. If you used to perform inline modifications such as <b>&apos;hello&apos; at: 1 put: $y</b> or <b>\#(1 3 2) sort</b>, you will need to copy the literal first. Brace expressions are not affected.
- Identity forwarding via <a href="code://TextLink new classAndMethod: &apos;Object becomeForward:&apos;; actOnClickFor: Model new">\#becomeForward:</a> or <a href="code://TextLink new classAndMethod: &apos;Array elementsForwardIdentityTo:&apos;; actOnClickFor: Model new">\#elementsForwardIdentityTo:</a> does no longer copy the hash. Append <b>copyHash:</b> to the selector to override the default behavior.
- Tests that signal an unhandled <a href="code:// Warning">warning</a> will now be considered as errored, even if the warning does not raise a debugger.
- <a href="code:// ToolSet browseMessageNames: &apos;chooseFrom*&apos;">UIManager &gt;&gt; \#chooseFrom\*</a> will no longer present the values as buttons. When requesting a choice from a small number of options, clients should migrate to the new <a href="code:// ToolSet browseMessageNames: &apos;chooseFrom*&apos;">UIManager &gt;&gt; \#chooseOptionFrom\*</a> interface.
- <a href="code://TextLink new classAndMethod: &apos;Process terminate&apos;; actOnClickFor: Model new">Process &gt;&gt; \#terminate</a> will now consistently execute all unwind contexts which may include the top context of the process. This implies that in rare cases sending \#terminate to a process will not interrupt the current control flow immediately.
- <a href="code://TextLink new classAndMethod: &apos;Process suspend&apos;; actOnClickFor: Model new">Process &gt;&gt; \#suspend</a> will no longer unblock a process that has been waiting for a semaphore or a mutex, but return to the blocked state after being <a href="code://TextLink new classAndMethod: &apos;Process resume&apos;; actOnClickFor: Model new">\#resume</a>d. To manually unblock a process indeed, send <a href="code://TextLink new classAndMethod: &apos;Process suspendAndUnblock&apos;; actOnClickFor: Model new">\#suspendAndUnblock</a> instead.
- In <a href="code://SyntaxErrorNotification">SyntaxErrorNotification</a>, \#resume: was renamed to <a href="code://#retryWithNewSource:">\#retryWithNewSource:</a> to clarify that the control flow will not just resume but compilation will be retried.
- <a href="code://#asMutator">\#asMutator</a> is back to preserve the well-known Smalltalk legacy and improve the backward compatibility for projects such as Seaside. However, new code should use <a href="code://#asSimpleSetter">\#asSimpleSetter</a> instead.
- The default value for <a href="code://#allowBlockArgumentAssignment">\#</a><a href="code://#allowBlockArgumentAssignment">llowBlockArgumentAssignment</a> is now enforced, which means <i>disabled</i>. Your project might have to enable this preference again if you want to assign to block arguments.
- Legacy US-layout-specific keyboard shortcuts for enclosing the selection with brackets have been disabled by default. They are still available through the <a href="code:// (Preferences pragmaPreferenceFor: LegacyShortcutsFilter getter: #legacyShortcutsEnabled) openInCategory: &apos;editing&apos;">Legacy keyboard shortcuts</a> preference. For all default shortcuts, read the <a href="code:// HelpBrowser openOn: (SqueakTutorialsCommandKey asHelpTopic subtopicAt: #commandKeyMappings)">help</a>.
